# Hermes Project Context — KTMB Pulse

## Project Overview

KTMB Pulse is a crowd & trend forecast app for Malaysia's railway network (KTMB). It spans two platforms:
- **Android app** (`android/`) — Kotlin + Jetpack Compose, 6 screens (Home, Forecast, Live, Stations, Alarms, Info)
- **HTML prototype** (`docs/`) — Dark premium theme with multi-tab navigation, SVG Malaysia map

## Working Rules for Hermes

### Safety Boundaries
- **Never touch `docs/data/`** — auto-generated by build pipeline; any changes will be overwritten
- **Never touch `android/app/src/main/assets/data/`** — bundled data mirrored from docs/data/
- **Never delete Gradle files, AndroidManifest.xml, Kotlin source, or app resources**
- **Never delete `.github/workflows/`** unless the workflow is broken and a replacement is provided
- **Never modify `.gitignore`** to un-ignore files without explicit user approval
- **Do not push** unless explicitly asked

### Public Repository Rules
- **No secrets shall be committed** — check all files for API keys, tokens, passwords before adding
- **Generated/build files must stay gitignored** — see `.gitignore` for current rules
- **AI-generated mockup images (Firefly*) are gitignored** — they are design exploration, not source
- **ZIP files are gitignored** — use individual source files, not archives
- **Screenshots are gitignored** — they're ephemeral evidence, not project assets

### What Files Hermes Must Not Edit Carelessly
- `android/` — the full Android app (Gradle, Kotlin, Compose, assets, manifest)
- `docs/data/` — generated JSON (station data, forecasts, routes, bbox)
- `docs/vendor/` — vendored third-party files (protobuf schema)
- `scripts/` — data pipeline Python scripts
- `src/` — Cloudflare Worker source
- `.github/workflows/` — CI/CD pipelines

### How Hermes Should Report Changes
- After every file modification, show `git status`
- After deletions, confirm no build breakage (run Gradle if possible)
- Report any failures honestly — do not hide or paper over errors
- List files created, deleted, and intentionally kept

### Deletion Candidate Handling
- Categorize into A (safe), B (needs confirmation), C (keep), D (security concern)
- Delete only Group A
- For Group B, list as candidates and explain the risk of deleting vs keeping
- For Group D, redact secrets and create a remediation note

### Model/Cost Discipline
- Use cheaper/faster models for simple edits, file operations, and bulk deletions
- Use stronger models for architecture decisions, risky changes, and complex Kotlin refactors
- Batch independent operations (reads, searches, deletes) in parallel

## Data Architecture

| Source | Format | Used |
|--------|--------|------|
| `komuter_2025.parquet` | Parquet | ✅ Forecast |
| `komuter_2024.parquet` | Parquet | ❌ (available for YoY) |
| `komuter_utara_2025.parquet` | Parquet | ✅ Forecast |
| `komuter_utara_2024.parquet` | Parquet | ❌ (available for YoY) |
| `ets_2025.parquet` | Parquet | ❌ UNUSED (496K rows) |
| `intercity_2025.parquet` | Parquet | ❌ UNUSED (243K rows) |
| GTFS-Realtime | Protobuf | ✅ Live vehicle positions |

Total: ~8.3M ridership records, 153 unique stations, 4 services.

## Data Pipeline
- `scripts/build_data.py` — Builds `docs/data/*.json` from parquet sources
- `scripts/build_gtfs_routes.py` — Builds route polylines + bbox from GTFS static
- `.github/workflows/build-data.yml` — Scheduled workflow commits updated data
- JS data: `docs/data/by_origin/{slug}.json` — Per-origin hourly crowd forecasts

## Live Feed
- `src/index.js` — Cloudflare Worker proxy for GTFS-RT vehicle positions (adds CORS)
- `wrangler.jsonc` — Worker deployment config

## Android App Structure
- Package: `com.ktmb.crowdtrend`
- Gradle kts, Kotlin/Compose, MVVM with DataStore
- 6 screens: Home → Forecast → Live → Stations → Alarms → Info
- Forecast uses `build_data.py` output baked into assets
- Live: 30s auto-poll via GTFS-Realtime API
- TransitAlarmService: foreground service with GPS proximity alerts
- Uses `tripUpdateCache` + `vehiclePositionCache` for live state
- StationData includes: names, states, GTFS stop IDs, services per station

## Build/Test Commands
```bash

cd android && ./gradlew assembleDebug --no-daemon

# Alternative via PowerShell (if on Windows host)
powershell -File _build_apk.ps1

# Python data pipeline
cd scripts && python build_data.py
cd scripts && python build_gtfs_routes.py
```
