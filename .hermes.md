# Hermes Project Context — KTMB Pulse

## Project Overview

KTMB Pulse is a crowd & trend forecast app for Malaysia's railway network (KTMB). It spans two platforms:
- **Android app** (`android/`) — Kotlin + Jetpack Compose, 4 screens (Forecast, Live, Stations, Settings)
- **HTML prototype** (`docs/`) — Dark premium theme with 5-tab navigation, SVG Malaysia map

## Data Architecture

| Source | Format | Used |
|--------|--------|------|
| `komuter_2025.parquet` | Parquet | ✅ Forecast |
| `komuter_2024.parquet` | Parquet | ❌ (available for YoY) |
| `komuter_utara_2025.parquet` | Parquet | ✅ Forecast |
| `komuter_utara_2024.parquet` | Parquet | ❌ (available for YoY) |
| `ets_2025.parquet` | Parquet | ❌ UNUSED (496K rows) |
| `intercity_2025.parquet` | Parquet | ❌ UNUSED (243K rows) |
| GTFS-Realtime | Protobuf | ✅ Live vehicle positions |

Total: ~8.3M ridership records, 153 unique stations, 4 services.

## Data Pipeline
- `scripts/build_data.py` — Builds `docs/data/*.json` from parquet sources
- `scripts/build_gtfs_routes.py` — Builds route polylines + bbox from GTFS static
- `.github/workflows/build-data.yml` — Scheduled workflow commits updated data
- JS data: `docs/data/by_origin/{slug}.json` — Per-origin hourly crowd forecasts

## Live Feed
- `src/index.js` — Cloudflare Worker proxy for GTFS-RT vehicle positions (adds CORS)
- `wrangler.jsonc` — Worker deployment config

## Android App Structure
- Package: `com.ktmb.crowdtrend`
- Gradle kts, Kotlin/Compose, MVVM with DataStore
- Screens: Forecast → Live → Stations → Settings
- Forecast uses `build_data.py` output baked into assets
- Live: 30s auto-poll via GTFS-Realtime API
- Uses `tripUpdateCache` + `vehiclePositionCache` for live state
- StationData includes: names, states, GTFS stop IDs, services per station
